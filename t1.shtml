Tutorial 1

    Set up github + p5.js + your homepage

    1. Create your GitHub account
    Go to github.com and sign up for an account. Using your UNI keeps things simple but you're free to use whatever you like.
    Use default settings and click Continue.
    Answer survey questions as you like, and click Submit.
    Once on this screen, hit Start a project.

    2. Make your class website project + install GitHub client
    Type your repository name exactly as yourusername.github.io. (If this is not exact, you will run into problems later.) Enter a description that makes sense to you, keep the rest of the settings as in, and click Create repository.
    Once on this screen, click Set up in Desktop.
    You'll be brought to this page; click on Download for macOS (or whatever OS you have.)
    Install the GitHub Client as a normal application. If on Mac, you should have it in your Applications folder like so.
    Open the GitHub client to see this screen. Click on Sign in to GitHub.com.
    Enter your username and password.
    Confirm your email if prompted.
    Next, you'll be prompted to "clone a repository." Select the yourusername.github.io project you made earlier, make note of where the Local Path folder is on your computer, and click on the Done button.
    Your GitHub Client screen should now look like this.
    Find the folder that was in Local Path. By default on a Mac, this is /Documents/GitHub/username.github.io.

    3. Copy p5.js starter example
    Next, visit the P5.js Download page and grab the p5.js complete files.
    Unzip the downloaded file to see a folder called p5.
    Update Jan.21.2018: Please download this sketch.js file and replace the one in your /empty-example folder before proceeding.

    Open the p5 folder and open the index.html file under empty-example folder.
    The file should open in your default browser and show an animated p5.js sketch that looks something like this.
    Now we're going to copy the starter ("empty") example to your GitHub project folder.
    Select the four files: p5.dom.min.js, index.html, sketch.js, p5.min.js and copy them into your username.github.io folder.
    Your /username.github.io folder should now look like this.
    Open /username.github.io/index.html, and this time it should not work: you should see a blank screen like this.
    Now visit Sublime Text to download and install the Sublime code editor. (If you already have a favorite code editor, feel free to skip this step.)
    Once you have Sublime installed, open /username.github.io/index.html with Sublime.
    You should see this. We're going to edit lines 5-8...
    ...to look like this. (Yes, delete line 8 from the original.) Hit Save.
    Now if you reload the /username.github.io/index.html file in your browser, you should see the starter example working again.

    4. Upload (publish) starter example to your class website
    Next we're going to put the starter example at your personal url. Go back to the GitHub Client and you should now automatically see this. (It reflects the files we copied to your project folder.) In the bottom left, locate the Summary field.
    Type a message that summarizes the changes we just made (you don't have to match mine exactly), and click on Commit to Master.
    Your screen should now look like this. Click on Publish branch located in the top right.
    Observe the message changing as the files on your computer are sent to the GitHub servers...
    You will know uploading is complete when the button automatically says Fetch origin, Last fetched just now.
    Now open a new browser tab and go to https://github.com/yourusername/yourusername.github.io and you should see this screen, listing the files you uploaded and your summary message.
    Within 5 minutes, you (or a friend) can open https://yourusername.github.io and voila! You've just published the starter example at your own url.

    5. Edit starter example
    Now we're going to edit the starter example and republish. Find the /username.github.io/ folder on your computer again, and this time open up sketch.js in Sublime.
    You should see this.
    Select lines 1-12 and go to Edit > Comment > Toggle Comment.
    You should see lines 1-12 change from grayed out to colored JavaScript code.
    Now delete everything after line 12 and you should have this. Hit Save.
    Now refresh your /username.github.io/index.html file in your browser and you should see this when you hover your mouse over the top left area of the window.

    6. Change example to your A1 assignment
    Now we're going to walk through your A1 Helloworld assignment. Go back to your /username.github.io/sketch.js file in Sublime, and try out some lines of code from the p5.js reference, such as text() to draw a word, fill() to change colors, rect() to draw a rectangle, and ellipse() to draw a circle, and background() to set a background color.
    One thing you may have noticed is that any line that started with two slashes, like so //, becomes grayed out and does not affect the output. These are called "comments" and you're free to use them to annotate your code.     7. Publish your A1 assignment
    Now we're going to publish your A1 changes via the same steps we took to publish the circle-follow-mouse sketch earlier. Return to your GitHub client, and this time, because we removed so many lines of code, most of the screen appears in red. Locate again the Summary field.
    Enter a summary that makes sense to you and hit Commit to master.
    Now hit the Push Origin button in the top right.
    You can double-check that everything is successfully uploaded by refreshing your https://github.com/username/username.github.io url. You should see the summary line next to sketch.js reflect the summary you last entered.
    Now refresh your browser at https://username.github.io and you should see your A1 assignment published on the web. (If it doesn't update the first time, provided everything looks correct at https://github.com/username/username.github.io, just wait a few minutes and try refreshing again.)

    Congrats you've completed Tutorial 1.
